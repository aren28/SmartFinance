# 設計決定事項

## データモデル設計

### amount: Int（円単位固定）

- 小数点を持たない整数で金額を扱う
- 理由: 日本円は事実上円単位で取引される。Float では浮動小数点誤差が生じる可能性がある
- ¥150.5 のような入力はAIが四捨五入して整数化する

### rawText 保持

- ユーザーが入力した元のテキストをそのままDBに保持する
- 理由:
  1. AI解析が失敗・誤った場合の再解析用
  2. デバッグ時にAIへの入力を確認できる
  3. 将来的な解析ロジック改善時のリトレーニング素材

### カテゴリ削除: onDelete Restrict

- 支出が紐づいているカテゴリは削除不可
- 理由: 孤立したレコードによるデータ不整合を防ぐ
- 対応: UI側でも `_count.expenses > 0` の場合は削除ボタンを disabled にする（二重防護）

## AIモデル選択

- **Claude Haiku** を使用（claude-haiku-4-5-20251001）
- 理由: 支出解析・アドバイス生成はシンプルなタスクであり、Haikuで十分な精度が得られる。コスト・レイテンシを最優先

## データベース

- **Supabase (PostgreSQL)** を使用
- 接続プール経由 (pgBouncer): `DATABASE_URL`
- 直接接続 (マイグレーション用): `DIRECT_URL`

## エラーハンドリング方針

- Prisma エラーコードを判定し、ユーザー向けの日本語メッセージに変換
- `P2003` (FK違反) → 409 + 削除不可メッセージ
- `P2002` (UNIQUE違反) → 409 + 重複メッセージ
- `P2025` (レコード未発見) → 404
- その他 → 500 + 詳細はサーバーログのみ

## カテゴリ自動生成

- AIが解析したカテゴリ名で `upsert` する
- 既存カテゴリが存在すれば再利用、なければ作成
- デフォルトカテゴリ: 食費/交通費/娯楽費/日用品/医療費/その他

## 画面構成

### ホーム画面（シングルページ、390×844px）
- **AIアドバイスバナー**（起動時表示）
  - 背景: #E1F5EE / タグ背景: #9FE1CB / 角丸14px
  - 今月の傾向・節約アドバイスを表示
- **今月の支出合計カード**
  - 金額（太字28px）+ 前月比バッジ（▲ 15% など）
  - バッジ背景: #FCEBEB / 文字: #A32D2D
- **テキスト入力欄**
  - カテゴリ選択チップ（食費・交通費・カフェ・日用品）
  - プレースホルダー「例：昨日ランチ 800円」
  - 送信ボタン（#1D9E75、34×34px）
- **支出履歴リスト**
  - カテゴリバッジ + 名前 + 時間 + 金額 + 削除ボタン

### 設定画面（390×844px）
- **カテゴリ管理**
  - カテゴリ一覧（丸いカラーアイコン + 編集 / 削除ボタン）
  - カテゴリカラー例: 食費 #7F77DD / 交通費 #378ADD / カフェ #BA7517 / 日用品 #D4537E / 娯楽 #639922
  - 「＋ カテゴリを追加する」（破線ボーダーボタン）
